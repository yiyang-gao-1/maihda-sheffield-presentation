How MAIHDA Changes Everything

5/13

The MAIHDA Model

CONVENTIONAL:

schools ~ period + controls

VS

MAIHDA:

schools ~ period + (period | strata_id) + (1 | household_id)

The key term (period | strata_id) allows policy effects to vary by intersectional group!

What MAIHDA Reveals:

✓ Policy works GREAT for:

  • White British families (78% take-up)
  • Those with cars
  • Faith-based users

✗ Policy FAILS for:

  • Pakistani families (42% take-up)
  • Black families at 2.5mi (0% - not eligible)
  • Those facing information barriers

🎯 The Real Insight:

The "successful" average comes from high-performing groups compensating for failing groups. The policy is actually deepening inequalities - helping those who already have advantages while failing those who need it most!

The "Monte Carlo" Simulation Code

8/13

What the R code actually does:

mc_scenarios <- map_df(1:n_sims, function(i) { # Scenario A: Reduce threshold to 2.5 miles scenario_a <- households %>% mutate( eligible_a = distance_to_school >= 2.5, uses_a = rbinom(n(), 1, ifelse(eligible_a, takeup_prob, 0)), access_a = ifelse(uses_a, current_access + rpois(n(), 3), current_access) ) # ... similar for Scenarios B and C ... })

Step 1: Change eligibility

distance >= 2.5

Simple threshold change

Step 2: Random usage

rbinom() with existing take-up rates

But rates are predetermined!

🚨 The simulation runs 1,000 times but the core relationships never change!

What's Predetermined

9/13

1. Take-up rates are baked in from the start:

# In the data generation: base_takeup = case_when( ethnicity == "Pakistani" & eligible ~ 0.42, # Hardcoded! ethnicity == "White_British" & eligible ~ 0.78, # Hardcoded! ... )

2. The "improvement" in Scenario B is predetermined:

takeup_improved = case_when( ethnicity == "Pakistani" & eligible ~ pmin(0.71, takeup_prob * 1.7), eligible ~ pmin(0.9, takeup_prob * 1.2), )

I just multiply Pakistani take-up by 1.7 to get 71%!

3. The inequality measure is just the standard deviation:

inequality = sd(households$post_policy_access)

What the Monte Carlo Actually Does

10/13

The 1,000 simulations only add random variation to:

1. Individual Usage

Whether each person uses transport

rbinom(n(), 1, takeup_prob)

Binomial random draw

2. School Access

How many extra schools they access

rpois(n(), 3)

Poisson random draw

But the core relationships are all predetermined:

  • Scenario A will always help more people but less effectively
  • Scenario B will always help fewer people but more effectively
  • Scenario C will always fall in between

The Monte Carlo is essentially just adding noise to make the results look realistic!

The Real Value: Why This Is Actually Fine

11/13

This is fine for a methodological demonstration!

The point isn't to prove that outreach works better in London (we don't have real data).

The point is to show:

1. HOW MAIHDA would reveal differential impacts

The (period | strata_id) term allows effects to vary by group

2. WHY understanding intersectional barriers matters

Average success can mask group-specific failures

3. WHAT KIND of insights this method provides

Targeted interventions can be more effective than broad expansions

For Methodology Demonstration:

Using synthetic data with plausible patterns is standard practice. It allows us to clearly demonstrate analytical techniques without the noise and complications of real data.

The Beauty of Methodological Demonstrations:

We can clearly show HOW the method works without getting bogged down in:

  • Data quality issues
  • Missing values
  • Measurement error
  • Complex real-world confounders

Think of it like learning physics with frictionless surfaces and perfect spheres - simplified to show the core principles!
